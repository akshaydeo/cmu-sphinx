Provides a mechanism for managing persistent configuration data.
  • Minimize changes to code
  • Simplify property naming by eliminating the need for fully qualified property names
  • Retain the ability to easily find where a property is defined and used in the code
  • Define an alternate format properties file that will be a joy to use (comparitively speaking)
  • Allow Sphinx components to be notified when a particular property has been changed
  • Allow Sphinx components to reject property changees
  • Provide a mechanism to allow property changes from code
  • Provide a set of properties manipulators that will allow development of tools such as the following:
    • Property dumper - shows the properties in use for a particular component or for the whole system
    • Property editor - allows live editing of the properties
