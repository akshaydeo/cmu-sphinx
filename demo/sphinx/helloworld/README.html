Sphinx-4 Hello World! Demo

This is a simple command line program that shows how to write a simple Sphinx-4 application that recognizes connected digits, using a BNF-style grammar. You can run it in two modes: with or without endpointing. When you run the demo without endpointing, the following will appear: 

Say any digit(s): e.g. "two oh oh four", "three six five".
Start speaking. Press ENTER when you finish speaking.

You said: one two three four

Start speaking. Press ENTER when you finish speaking.

You said: four two one one four

Start speaking. Press ENTER when you finish speaking.

It expects you to say a sequence of digits, and press ENTER when you finish speaking. The recognition results will then be shown, and you will be prompted to say another sequence of digits. When you want to quit the demo, simply press Ctrl-C.

When you run the demo with endpointing, you do not have to press ENTER when you finish speaking. The endpointer will automatically detect the end of an utterance: Say any digit(s): e.g. "two oh oh four", "three six five". 

Say any digit(s): e.g. "two oh oh four", "three six five".
Start speaking. Press Ctrl-C to quit.

You said: one two three four

Start speaking. Press Ctrl-C to quit.

You said: four five six nine

Start speaking. Press Ctrl-C to quit.

Building

To build this demo, merely type the following in a shell from any directory in the Sphinx-4 hierarchy:

    ant -find demo.xml

Doing so will create bld/bin/HelloWorld.jar and bld/bin/HelloDigits.jar under the top level directory. You can run the demo using java -jar (see "Running" below).

Running

First make sure that you have JSAPI setup correctly. Then, make sure that all the Sphinx-4 classes are built. Go to the top level directory (sphinx4) and type:

ant all

To run the demo without the endpointer, go to the top level directory and type:

sphinx4> java -jar bld/bin/HelloWorld.jar

To run this demo with the endpointer, go to the top level directory and type:


sphinx4> java -jar bld/bin/HelloDigits.jar

NOTE: Make sure that you are using JavaTM 2 SDK, Standard Edition, v1.4 or higher.
